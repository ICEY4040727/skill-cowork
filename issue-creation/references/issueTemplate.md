# Issue 模板

用于生成可直接粘贴到 GitHub Issue 的最终正文。

## 直接粘贴版

```markdown
## 标题
[动词 + 对象 + 场景]

## 背景
[当前问题或目标]
[影响范围]

## 验收标准
- [ ] [可验证结果 1]
- [ ] [可验证结果 2]

## 技术说明
- 涉及模块：frontend / backend / both
- 数据库变更：是 / 否
- 依赖任务：#N 或无

## 优先级
P0 / P1 / P2
```

## 最小填写要求

- 标题不超过 20 个中文词，避免模糊词（如“优化一下”）。
- 验收标准至少 2 条，并且可被测试或人工核对。
- 技术说明必须填写依赖关系（无依赖也要写“无”）。
- 优先级只保留一个值。

## 建议附加内容（可选）

- 风险说明：上线风险、回滚策略。
- 参考链接：设计稿、日志、讨论串。

## Labels

| Label | 用途 |
|-------|------|
| `feature` | 新功能 |
| `bugfix` | Bug 修复 |
| `research` | 技术调研/提案 |
| `creator` | Creator 负责实现 |
| `reviewer` | Reviewer 提出/负责 |
| `P0` | 紧急 |
| `P1` | 重要 |
| `P2` | 一般 |
| `approved` | Owner 已批准 |
| `needs-review` | 待 Reviewer 审查 |
